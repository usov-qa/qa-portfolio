# Проект 5: Тестирование API Яндекс.Прилавка

Учебный проект в Яндекс Практикуме.  
Тестировал бэкенд через Postman: работа с наборами, доставкой и корзиной.

## Что тестировал
- Добавление продуктов в наборы: `POST /api/v1/kits/{id}/products`
- Расчёт доставки службой «Привезём быстро»: `POST /fast-delivery/v3.1.1/calculate-delivery.xml`
- Получение списка продуктов в корзине: `GET /api/v1/orders/{id}`
- Добавление продуктов в корзину: `PUT /api/v1/orders/{id}`
- Удаление корзины: `DELETE /api/v1/orders/{id}`

## Что делал
- Изучил требования и документацию к API в Apidoc
- Провёл декомпозицию требований, разбил на атомарные проверки
- Применил классы эквивалентности и граничные значения для времени, веса и количества продуктов
- Составил чек-лист для покрытия всей переданной функциональности
- Провёл тестирование API через Postman
- Оформил баг-репорты по найденным дефектам
- Написал отчёт о тестировании

## Отчёт о тестировании

Тестирование было непростым — самый сложный проект для меня на данный момент. Приходилось переключаться между Apidoc, Postman, требованиями к бэкенду и Google-таблицами, следя за синтаксисом запросов. Проект сложный, но не менее интересный.

Сначала я провёл декомпозицию требований, разбил их на атомарные проверки. Методом классов эквивалентности и граничных значений выявил проверки для времени заказа, веса и количества продуктов. Проверил валидность различных id (id продукта, количество продукта) и выявил несколько багов.

Тестировать API мне понравилось — думаю, со временем качество и скорость работы значительно вырастут.
